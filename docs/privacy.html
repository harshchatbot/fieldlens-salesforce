Overview

FieldLens for Salesforce is a Chrome extension that helps users analyze Salesforce Lightning field impact by scanning Salesforce metadata references (e.g., Apex, Flows, Validation Rules, Layouts, and FLS).

Data We Access

  • Salesforce page context (e.g., the current Lightning URL to identify object/field context).
  • Salesforce metadata responses returned from Salesforce REST/Tooling APIs for scanning dependencies.

FieldLens does not read Salesforce record data for tracking or analytics purposes.

Data We Store

  • Local settings (e.g., UI preferences, default scan mode).
  • Short-lived local cache (optional) to improve performance for repeat scans.

All storage is local to your browser using chrome.storage.

Session & Authentication

FieldLens uses your active Salesforce browser session to make authorized metadata requests to your Salesforce org. In some edge cases, FieldLens may read the Salesforce session cookie named sid using the Chrome cookies permission only as a runtime authentication fallback.

  • The sid cookie is not stored by FieldLens.
  • The sid cookie is not logged or sent to any external server.
  • The cookie is used only to authorize requests to the same Salesforce domain you are actively using.

Data Sharing

FieldLens does not sell, rent, or share user data with third parties. FieldLens does not transmit scan data to external servers. All processing occurs locally and via Salesforce APIs within your Salesforce session.

Remote Code

FieldLens does not use remote code. All JavaScript executed by the extension is packaged within the extension.
